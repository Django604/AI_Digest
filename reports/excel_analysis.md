# Excel 分析报告

源文件：`data/source/dashboard-source.xlsm`

## 1. 工作簿结构

仓库内的源工作簿副本现有 8 张表：

- `NEV全车系线索日趋势`
- `ICE全车系线索日趋势`
- `参数`
- `目标竖版`
- `全国按日NEV`
- `全国按日ICE`
- `十五代轩逸按日`
- `早报快捷模板`

其中真正驱动 Web 面板的数据链路是：

- `参数`：报表日期、上期日期、年/月/日参数
- `全国按日NEV`：NEV 日级明细事实表
- `全国按日ICE`：ICE 日级明细事实表
- `十五代轩逸按日`：ICE 子车型日级明细事实表
- `目标竖版`：NEV 车型按日目标表
- `NEV全车系线索日趋势` / `ICE全车系线索日趋势`：展示层面板

## 2. 公式引用关系

### `NEV全车系线索日趋势`

主要依赖：

- `参数`：245 处
- `全国按日NEV`：248 处
- `目标竖版`：93 处
- 自身表内引用：341 处

公式逻辑分层：

1. `参数` 控制当前报表日期和上期日期
2. `全国按日NEV` 按 `日期 + 意向基准车系` 聚合出各车型上期/本期实绩
3. `目标竖版` 按 `日期 + 车型` 聚合出本期目标
4. 展示表再做累计、达成率、环比、摘要文案拼接

### `ICE全车系线索日趋势`

主要依赖：

- `参数`：133 处
- `全国按日ICE`：62 处
- `十五代轩逸按日`：62 处
- 自身表内引用：31 处

公式逻辑分层：

1. `参数` 控制日期轴
2. `全国按日ICE` 汇总出 ICE 全车系上期/本期有效线索
3. `十五代轩逸按日` 单独拉出子车型趋势
4. 展示表完成累计、环比与摘要文案拼接

注意：

- ICE 总盘没有有效的目标输入，`本期目标` 实际为 0，所以总盘达成率显示为 `-`
- `十五代轩逸` 子模块目标行是面板内直接维护的值，不来自独立目标表

## 3. 数据逻辑结构

### 输入层

- `参数`
  - `C2`：日报日期
  - `F2`：上期对应日期
  - `C4:C7`：当前年月日与当月首日
- `全国按日NEV`
  - 关键字段：`C 日期`、`D 意向基准车系`、`E 线索开发`
- `全国按日ICE`
  - 关键字段：`B 按日`、`G 有效线索量`
- `十五代轩逸按日`
  - 关键字段：`B 按日`、`G 有效线索量`
- `目标竖版`
  - 关键字段：`A 日期`、`B 车型`、`N 合计`

### 计算层

- 日期轴：根据 `参数` 自动生成上月同日和本月日期序列
- 日目标：从目标表按日期匹配
- 日实绩：从事实表按日期与车型匹配
- 累计目标 / 累计实绩：按 `<= 报表日` 累加
- 达成率：`累计实绩 / 累计目标`
- 环比：`累计实绩 / 上期累计实绩 - 1`

### 展示层

- 摘要文案
- 指标卡片
- 日趋势横向表
- 分车型子面板

## 5. 风险与异常

已发现异常：

- `NEV全车系线索日趋势`
  - `#VALUE!` 30 个
  - `#N/A` 124 个
  - `#REF!` 1 个
- `ICE全车系线索日趋势`
  - `#N/A` 50 个
  - `#VALUE!` 1 个

这些异常里有两类：

- 未来日期故意返回空值或 `NA()`，属于展示用控制逻辑
- 真正坏引用，例如 `NEV` 里 `NX8` 预售分析区域

原始模板中的 `NEV大区汇总` 已从仓库内副本移除，避免废表继续污染后续流程。

## 6. 技术结论

- 这份 Excel 本质是“事实表 + 目标表 + 参数表 + 展示表”的典型多层结构
- Web 化时应该保留数据源与展示层分离
- `Django` 不适合直接上 `GitHub Pages`
- 当前项目采用“静态站点 + Excel 抽取脚本 + Pages 自动发布”是最省事也最稳的方案
